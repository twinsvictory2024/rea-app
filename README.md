# E-Commerce Platform (Дипломный проект)

Многомодульное веб-приложение для организации электронной коммерции, объединяющее продавцов и покупателей на одной платформе.

## 📋 Описание проекта

Платформа предоставляет полноценный функционал для ведения электронной коммерции:

- **`Продавцы`** могут регистрироваться, создавать магазины и управлять товарной номенклатурой
- **`Покупатели`** получают возможность находить товары, формировать заказы и совершать покупки

## 🏗 Архитектура

Проект построен на модульной архитектуре, что обусловлено спецификой e-commerce решений:

- Широкий функционал
- Большое количество моделей, роутов и представлений
- Сложная бизнес-логика

Такой подход позволяет с самого начала заложить основу для масштабирования и поддерживать чистоту кода на протяжении всего жизненного цикла проекта.

### Структура модулей

- **`rea/`** — настройки проекта и менеджер команд
- **`rea_auth/`** — модуль авторизации и аутентификации
- **`rea_catalog/`** — каталог товаров (продукты, категории)
- **`rea_common/`** — общие компоненты (миксины, утилиты, базовые модели)
- **`rea_eav/`** — EAV-атрибуты для гибкого описания товаров
- **`rea_orders/`** — управление заказами
- **`rea_users/`** — пользователи и контактная информация
- **`rea_vendors/`** — магазины продавцов

### Конвенции построения модулей

Каждый модуль следует единым стандартам организации кода. Помимо стандартных файлов Django/DRF, модуль может содержать:

- **`permissions.py`** — кастомные разрешения
- **`filters.py`** — фильтры для API
- **`tasks.py`** — Celery задачи для асинхронной обработки
- **`services/`** — сервисный слой с бизнес-логикой
- **`serializers/`** — раздельные сериализаторы для каждой модели


## 🛠 Технологический стек

- **`Backend`**: Django + Django REST Framework
- **`Асинхронные задачи`**: Celery + Redis
- **`База данных`**: PostgreSQL
- **`Аутентификация`**: Knox Token Authentication

## ⚙️ Установка и запуск

### Требования

- Python 3.12+
- Redis (для Celery)

### Настройка окружения

Проект поддерживает разделение сред разработки (development) и продакшена (production).

1. **Клонирование репозитория**

- git clone git@github.com:twinsvictory2024/rea-app.git
- cd rea-app

2. **Установка зависимостей**

- pip install -r requirements/development.txt

3. **Применение миграций**

- python rea/manage.py makemigrations
- python rea/manage.py migrate

4. **Запуск сервера разработки**

- python rea/manage.py runserver

5. **Запуск Celery worker (в отдельном терминале)**

- celery -A rea.config worker --loglevel=info
